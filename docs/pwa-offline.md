# Progressive Web App (PWA) & Offline Strategy

This document details the progressive web application configurations, build-time caching hooks, and install prompts that enable offline Quran reading.

---

## 1. Web App Manifest

The app's metadata is declared in `public/manifest.webmanifest`. Key fields include:

- `display: standalone`: Launches the app in a standalone window, hiding browser navigation bars to feel like a native app.
- `orientation: portrait`: Locks the UI to portrait mode for standard mobile reading.
- `theme_color`: Injected matching the dark theme background to style the status bar on mobile operating systems.
- `icons`: Defines icon bundles (e.g., `192x192` and `512x512`) used for home screen launches and the splash screen.

## 2. Service Worker Generation (Workbox)

Quran0 uses Workbox to cache production files. Because hashed asset names generated by Vite only exist after compilation, the service worker must be constructed after the final build.

### Nitro Compilation Hook

In `vite.config.ts`, the app hooks into the Nitro prerendering compiler's lifecycle:

```typescript
nitro({
  modules: [
    (nitroApp) => {
      nitroApp.hooks.hook('compiled', async () => {
        await generatePwa(nitroApp.options.output.publicDir)
      })
    },
  ],
})
```

### Build Script (`scripts/generate-pwa.ts`)

The generator runs `generateSW` from `workbox-build` to generate the production `sw.js`:

```typescript
const { count, size } = await generateSW({
  globDirectory: publicDirectory,
  globPatterns: [
    '**/*.{css,html,ico,js,json,otf,png,svg,ttf,webmanifest,woff,woff2}',
  ],
  globIgnores: ['sw.js', 'sw.js.map', 'workbox-*.js', 'workbox-*.js.map'],
  swDest: serviceWorkerPath,
  navigateFallback: '/index.html',
  cleanupOutdatedCaches: true,
  clientsClaim: true,
  skipWaiting: true,
  maximumFileSizeToCacheInBytes: 10 * 1024 * 1024,
})
```

### Caching Scope

The service worker precaches:

1. **Prerendered HTML Pages**: All 114 Surah pages (`/surah/1` to `/surah/114`) and primary routes (`/`, `/bookmarks`, `/progress`, `/memorize`, `/motivation`) prerendered into static HTML by TanStack Start.
2. **Lazy-loaded Verse Chunks**: Every generated JSON file (`src/data/quran/generated/surahs/*.json`).
3. **Static Assets**: Font files (`DigitalKhattIndoPak.otf`), stylesheet, logos, icons, and js scripts.

This ensures that once precached, the user can read the entire Quran offline, load pages, and access settings without any internet connection.

## 3. PWA Install Prompts (`src/components/pwa-install-prompt.tsx`)

Quran0 displays a custom, native-feeling installation prompt banner rather than relying on browser defaults.

### Flow & Features

- **Registration**: registers `/sw.js` in production mode.
- **Install Delay**: The prompt delays rendering by `1000ms` after page load to allow a clean first impression.
- **Dismissal Persistence**: Once dismissed, the prompt remains hidden for the remainder of the session using `sessionStorage` (`quran0-install-prompt-dismissed`).
- **Device Support**:
  - **Native**: Intercepts the `beforeinstallprompt` event on supported platforms (Chrome, Edge, Android) to display a single-tap "Install app" button.
  - **iOS/Safari**: Detects iOS devices (where `beforeinstallprompt` is unsupported) and renders custom directions instructing the user to tap the Safari "Share" icon followed by "Add to Home Screen".
